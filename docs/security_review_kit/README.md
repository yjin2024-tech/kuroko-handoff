# IT Security レビュー用メモ

このフォルダは、Kuroko Handoff の公開レビュー用の最小デモを、情シス・法務・業務部門が確認しやすい形に整理するための資料です。

このデモの目的は、生成AIや外部委託に渡す情報をそのまま増やすことではありません。お客様環境内でマスキングを行い、社外に出す情報を最小化したうえで、AI活用PoCや外部パートナーとの要件整理を進める流れを検証することです。

生成される `security_review.md` は、handoff package を外部パートナーへ共有する前に、社内のセキュリティレビューで確認するための資料です。共有してよいもの、ローカルに残すべきもの、その理由、そしてこの workflow が法的な匿名化・匿名加工情報ではない理由を説明します。

## このデモで確認できること

- Markdown、テキスト、CSV をローカルでマスキングする流れ
- 共有用 `handoff_package` と社内保管用 `local_only/mapping.local.json` を分けて管理する流れ
- 委託先に渡してよいファイルと、渡してはいけないファイルの分離
- マスキング済み資料を使って外部成果物を作成し、社内だけで復元する流れ
- マスキングレポートと確認キューによる人工確認ポイントの明示

## このデモで確認できないこと

- すべての個人情報や機密情報の自動検出
- 法令・社内規程への適合判断
- 本番環境での運用可否
- 端末管理、配布、監査ログ、署名付きリリース

本番利用を検討する場合は、社内承認フロー、委託先管理、データ保存期間、削除手順、利用するAIサービスの取扱範囲を別途確認してください。
