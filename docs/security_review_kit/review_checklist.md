# レビュー チェックリスト

## 情シス確認

- [ ] 実行端末と実行者が決まっている。
- [ ] 原文ファイル、`handoff_package`、`local_only/mapping.local.json` の保存場所が決まっている。
- [ ] `local_only/mapping.local.json` を委託先や外部AIに渡さない運用になっている。
- [ ] CLI のソースコード、依存ライブラリ、ビルド方法を確認できる。
- [ ] 本番利用前に、配布方法、署名、ハッシュ値、ログ方針を確認する。

## 法務・管理部門確認

- [ ] 外部委託先に渡す情報の取扱範囲が契約または発注書で明確になっている。
- [ ] 原文の個人情報を外部AIへ入力しない運用になっている。
- [ ] マスキング処理が法的な匿名化を意味しないことを関係者が理解している。
- [ ] 再委託、AIサービス利用、データ保存期間、削除方法が確認されている。

## 業務部門確認

- [ ] マスキング後の文章で、外部パートナーが作業できるだけの文脈が残っている。
- [ ] 伏字化しすぎて要件理解が困難になっていない。
- [ ] 業界固有の社名、案件名、略称をカスタム辞書に追加している。
- [ ] マスキングレポートを見て、残存リスクを人工確認している。
